# v0.5, hnscc-time-multimodal

**Arm 5 — Python↔R bridge deconvolution cross-check.** Demonstrates a clean
language-boundary integration and uses it to *cross-validate* the genomics arm.

## What ships

### Arm 5, R-bridge deconvolution
- New module `hnscc_time.deconv_r`: writes the cohort TPM matrix out, invokes a
  deterministic `Rscript --vanilla` subprocess, reads per-sample × cell-type
  scores back, and computes **Spearman agreement** against the Arm-2 ssGSEA
  scores (a pure, offline-testable scorer `cross_method_agreement`).
- New `scripts/deconv_markers.R`: **base R only** (no CRAN/Bioconductor install) —
  a marker **z-score** estimator. Deliberately a *different normalization* from
  the Python **mean-rank**, so the agreement check measures cross-method
  robustness, not a tautology. Marker panels mirror `genomics.IMMUNE_SIGNATURES`
  (CD3 / CD8 / FoxP3 / PanCK) so the two are comparable.
- New optional pipeline arm (Arm 5), audit-bracketed, that **skips cleanly when R
  is absent** (the same contract the IHC arm uses for the cellpose extra) or when
  the cohort has no STAR data.

## Honesty boundary (important)

- **No agreement number is claimed in this release.** The bridge runs only where
  an R interpreter is installed; it is **not exercised in CI or the build sandbox**
  (no R there), so the Spearman value is produced when the arm is run on an R host
  (e.g. a lab node) — not asserted here. The README table marks Arm 5 as a
  capability ("runs where R is installed"), not a result.
- What *is* verified offline: the cross-method scorer (`cross_method_agreement`)
  on synthetic frames (perfect-agreement, too-few-samples, zero-variance cases),
  and that the arm raises/skips correctly when `Rscript` is unavailable. An
  integration test runs the actual R script **only if `Rscript` is on PATH**
  (`@skipif`), so it executes on R hosts and is skipped elsewhere.
- The base-R estimator is a transparent marker z-score for **bridge + cross-method
  demonstration** — it is not a re-implementation of, or a claim to match, any
  published deconvolution package (MCP-counter, CIBERSORT, etc.).

## Measured (post-tag, on an R host — a local workstation, 2026-06-07)

Ran on a host with R installed (so the integration test executed, not skipped):

- **mean Spearman 0.96** across n=50 TCGA-HNSC (CD3 0.96 / CD8 0.97 / FoxP3 0.97 /
  PanCK 0.95). The Python↔R bridge round-tripped correctly and the base-R z-score
  and Python mean-rank rank samples near-identically. High agreement is expected
  (shared marker panels, different normalization) — it confirms **bridge integrity
  + normalization-robustness**, not agreement of two unrelated methods.
- `make data` reproduced every pinned checksum on the real host (50 STAR sha256,
  clinical `cebcec…`, hpv `76485138…`) and built `_subset_manifest.tsv` — i.e. the
  whole data layer is byte-reproducible end-to-end from the manifest.

## Reproducibility / tests

- No new Python dependency (uses `subprocess` + stdlib `shutil`).
- `tests/test_deconv_r.py`: scorer unit tests (offline) + skip-without-R behavior
  + an R-host-only integration test.
- `make run` gains Arm 5; on a host without R it records `arm5_skipped_no_R` in the
  audit ledger and continues.
