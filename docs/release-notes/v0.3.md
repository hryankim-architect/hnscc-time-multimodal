# v0.3 — hnscc-time-multimodal

First substantive public release. **Capability portrait, not a research result.**
v0.1, v0.2, and v0.3 all landed in a single Sunday-evening session
(2026-05-24); the table below documents what each planned tag would have
delivered if the sprint had run across the originally-budgeted week.

## What ships

Three-arm multimodal pipeline staged on the shared scaffold substrate:

### Arm 2 — Genomics (originally v0.1)
- Real TCGA-HNSC n=50 STAR-counts subset (GDC API, open tier)
- ssGSEA-style per-sample scoring against curated immune signatures
  (CD3 / CD8 / FoxP3 / PanCK; well-known canonical markers, no
  controlled-access signature matrices)
- Z-score cohort normalisation + per-patient TIMEProfile JSON in
  the common schema

### Arm 1 — IHC (originally v0.2)
- Cellpose nuclei segmentation (CPU mode, no GPU required)
- 5 real DeepLIIF Sample_Large_Tissues HNSCC ROIs
- Per-cell marker-channel intensity aggregation -> per-ROI
  TIMEProfile JSON
- Documented v0.2 heuristic R/G/B -> PanCK/CD8/CD3 mapping
  (placeholder pending PMC10571229 multi-channel mIF archive URL
  resolution — see `data/manifest.yaml`)

### Arm 3 — Cross-cohort calibration (originally v0.3)
- Nearest-neighbor matching from each IHC profile into the
  TCGA-HNSC genomics cohort
- Per-cell-type linear calibration mapping
- Leave-one-out validation reporting LOO MAE vs intercept-only
  baseline (honest "is calibration actually adding signal" comparison)
- `predict_time_from_genomics(genomics_profile) -> TIMEProfile`
  callable as the deployment-ready unit of work

## Substrate contract preserved

Same four-channel substrate as P1 / P2 / P3:
- `hnscc_time.audit` — hash-chained NDJSON ledger, every arm emits
  `cohort.assembled` / `time_profiles.computed` / `calibration.trained`
- `hnscc_time.tracking` — MLflow with no-op fallback
- `hnscc_time.canary` — daily probe consumed by `lab_semantic_check.py`
- English-only public surface enforced by CI scanner

## Honest scope (v0.3)

- **n=5 IHC reference** is one order of magnitude below what the full
  PMC10571229 archive (72 ROIs across 8 patients) would provide;
  calibration is a *demonstration of the integration pattern*, not a
  research finding.
- **Sample_Large_Tissues are RGB composites**, not true 4-channel mIF;
  Arm 1 uses a documented heuristic R/G/B -> marker mapping that Arm 3
  calibration then anchors against the genomics signal.
- **Cross-cohort, not paired-patient** — PMC8 patients are not in
  TCGA-HNSC; Approach B (calibration via nearest-neighbor matching) is
  used openly, as documented in `docs/architecture.md` and `~/Downloads/AI/P4-IHC-Genomics-TIME-Plan.md`.

## Reproducibility

- `make test` exercises 34+ unit tests (more if `[ihc]` extra installed)
- `make run` end-to-end pipeline on real TCGA-HNSC subset + 5 ROIs
- Hash-chained audit ledger spans `pipeline_start` -> per-arm emits ->
  `pipeline_end`; `audit.verify()` walks the chain and confirms tamper
  detection (covered in the test suite)
- English-only CI scanner enforces a CJK-free public surface

## What v0.4+ would add

- Resolve PMC10571229 multi-channel mIF archive URL -> swap Arm 1's
  RGB heuristic for true per-channel marker classification
- HPV+ vs HPV- subgroup analysis (TCGA-HNSC has HPV status; substrate
  is sized for it)
- Full ~530-patient TCGA-HNSC run (current n=50 is sufficient for the
  capability demo)
- R bridge for CIBERSORTx / MCPcounter cross-tool agreement
